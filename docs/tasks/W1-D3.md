# W1-D3 — Camera Follow + Graybox Map

> 必读：`docs/tasks/CONTEXT.md`、确认 D2 完成（玩家能走）

## 目标
摄像机跟随玩家；地图变成 30×30 瓦片的灰盒地图，墙能挡住玩家。

## 必读文件
- `src/scenes/player.tscn`（D2 创建的）
- `src/scenes/main.tscn`（要扩展）

| `src/scenes/player.tscn` | 修改 | 加 Camera2D 子节点 |
| `src/scenes/raid.tscn` | 新建 | 灰盒 raid 场景 |
| `src/scenes/main.tscn` | 修改 | 改名 / 移除 player → 现在 player 在 raid 场景里 |

## 步骤

### 1. [CODE] 给 `player.tscn` 加 Camera2D

在 player 场景里加一个 Camera2D 节点作为子节点：

修改 `src/scenes/player.tscn`，加入 Camera2D：
```
[node name="Camera2D" type="Camera2D" parent="."]
position_smoothing_enabled = true
position_smoothing_speed = 8.0
zoom = Vector2(1.5, 1.5)
```

> Camera2D 默认就 enabled 当前；写成 player 子节点会自动跟随玩家。

## 常见坑

1. Camera2D 在 Godot 4 默认就 enabled = current — 不需要再设
2. `position_smoothing_*` 在 Godot 4 是 Camera2D 的属性，不是 SmoothingEnabled 节点
3. ColorRect 是 Control 节点，不是 Node2D — 在 Node2D 父节点下用它没问题但要注意 z 顺序
4. StaticBody2D 必须有 CollisionShape2D 才挡得住
5. 视口默认 1080×1920（竖屏），zoom 1.5 让玩家看到约 720×1280 的范围 — 适合手机
